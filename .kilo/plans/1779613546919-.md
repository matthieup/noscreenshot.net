# Visitor Counter Implementation Plan

## Goal
Add a visitor counter to show how many people have visited the "no screenshot" page to learn why screenshots are not good.

## Chosen Solution
Use a third-party counting service (CountAPI.xyz) for simplicity and no backend required.

## Implementation Steps

### 1. Service Selection
- **CountAPI.xyz**: Provides a simple API to hit and retrieve counts via JSON or JSONP.
- Endpoint: `https://api.countapi.xyz/hit/<namespace>/<key>` increments and returns new count.
- No registration needed for basic usage.
- Supports CORS for fetch/XHR.

### 2. HTML Modification
- Add a container element to display the count.
- Suggested location: inside the footer-card section (after the existing paragraphs) or in the hero section.
- Example: `<div id="visitor-count" class="visitor-count">Visitors: <span id="count">loading...</span></div>`
- Style the counter appropriately (small, muted text).

### 3. JavaScript Implementation
- Add a script tag before the closing `</body>` tag.
- On page load:
  - Fetch from `https://api.countapi.xyz/hit/noscreenshot.site/visitors`
  - Parse JSON response.
  - Update the DOM element with the count value.
- Error handling:
  - If fetch fails, show a fallback message or hide the counter.
  - Optionally retry or use cached value from localStorage as fallback.

### 4. Styling
- Add minimal CSS for the counter in `styles.css` or inline.
- Ensure it matches the site's aesthetic (muted color, small font).
- Possibly add class `.visitor-count` with appropriate styling.

### 5. Testing
- Verify counter increments on each page load (or at least shows a number).
- Check that the layout remains intact on various screen sizes.
- Ensure no console errors.

## Files to Modify
- `index.html`: Add counter container and script tag.
- `styles.css`: Add styles for the counter (optional).

## Considerations
- CountAPI may have rate limits; for low-traffic site this is fine.
- The count is per namespace/key; using `noscreenshot.site/visitors` ensures uniqueness.
- If the service goes down, the counter will fail gracefully.

## Future Enhancements
- Store last known count in localStorage to show immediately while fetching.
- Display total visits with formatting (e.g., 1,234).
- Add tooltip or title explaining the counter.
